SFX attendu : game_over
========================

Fichier      : game_over.wav (ou game_over.ogg, les deux sont supportes)
Emplacement  : auto-sokoban/assets/audio/game_over.wav
Reference    : joue dans ui/scenes/game_over.py (methode on_enter)

Contexte
--------
Ce son est declenche lorsque le joueur arrive sur la scene GAME OVER,
suite a un deadlock detecte (caisse bloquee contre un mur) ou a un
abandon manuel de la partie.

Il doit etre clairement distinct du son "win" (victoire) deja present
dans le dossier, afin que l'oreille differencie immediatement une fin
heureuse d'une fin ratee.

Contraintes techniques
----------------------
- Format       : WAV 16-bit PCM (ou OGG Vorbis)
- Frequence    : 22050 Hz ou 44100 Hz
- Canaux       : mono ou stereo (les deux marchent)
- Duree        : entre 0.4 s et 1.5 s (court, on ne veut pas bloquer
                 le joueur)
- Taille max   : ~150 Ko (largement suffisant pour 1.5 s en WAV 16-bit)
- Volume       : normalise autour de -6 dBFS (le AudioManager applique
                 ensuite un facteur de volume utilisateur)

Direction artistique suggeree
-----------------------------
- Sonorite retro / chiptune coherente avec les autres SFX du jeu.
- Glissando descendant, tonalite mineure, ou bruit blanc bref.
- Eviter les voix, eviter les sons trop longs ou trop aigus.

Sources CC0 possibles
- Bfxr (generateur chiptune)    : https://www.bfxr.net/
- The Motion Monkey retro pack  : https://www.themotionmonkey.co.uk/free-resources/retro-arcade-sounds/
- OpenGameArt CC0 SFX           : https://opengameart.org/content/cc0-sound-effects

Verification
------------
1. Placer le fichier dans auto-sokoban/assets/audio/
2. Lancer le jeu, provoquer un deadlock (ou quitter une partie)
3. Le son doit etre joue au moment ou la scene GAME OVER apparait
4. Aucun warning "SFX introuvable : game_over" dans la console
